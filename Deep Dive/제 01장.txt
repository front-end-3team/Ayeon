Deep Dive 제 01장 : 프로그래밍

1.1 프로그래밍이란?
    - 프로그래밍이란 컴퓨터에게 실행을 요구하는 일종의 ** 커뮤니케이션 **.
    - 기계가 실행할 수 있을 정도로 정확하고 상세하게 요구사항을 설명하는 작업.

    - 문제 해결 능력과 Computational thinking(컴퓨팅 사고) 필요.   

    - 문제 해결 능력은 직감과 직관의 영역이라고 볼 수 있음( = 우리의 사고와 경험에 영향 받음 )

    - 컴퓨터와 사람은 사고 및 인지 방식이 다르기 때문에 컴퓨터의 관점에서 문제를 사고해야 함.
        ( = 논리적 수학적 사고가 필요하고 해결 과제를 작은 단위로 분해, 패턴화해서 추출, 
                            프로그래밍 내에서 사용될 모든 개념은 평가 가능하도록 정의해야 함)


1.2 프로그래밍 언어
    - 문제 해결 능력을 바탕으로 정의된 문제 해결 방안 -> 컴퓨터에게 전달
    - 사람이 이해할 수 있는 자연어는 컴퓨터가 이해할 수 없기에, 이해 가능한 기계어로 명령 전달(machinde code)
    
    ** 컴파일러 compiler 혹은 인터프리터 interpreter 란?
    - 프로그래밍 언어를 사용해 프로그램 작성 후, 컴퓨터가 이해할 수 있는 기계어로 변환하는 일종의 번역기

    - 프로그래밍 언어는 ** 구문 syntax ** 과 ** 의미 semantics ** 의 조합으로 표현


1.3 구문과 의미
    - 외국어(= 프로그래밍 언어)를 잘하려면 외국어 화자의 말이나 문장을 정확히 이해 후,
        문맥에 따른 적절한 어휘 선택, 그리고 순차적으로 결론을 향해 나아가는 문장 구성이 필요

    => 문법에 맞는 문장을 구성하는 것은 의미 semantics를 가지고 있어야 언어의 역할 충실히 수행

    ex) const number = 'string';
        console.log(number * number); // NaN
    
    - 자바스크립트(js)의 변수에는 어떠한 타입의 값도 할당 가능
    - 예제(ex)는 문법적으로 문제는 없으나 의미적으로는 옳지 않다 => number 라는 이름의 변수에 문자열이 할당되었기 때문.
                                                              => 옳은 방법 : 숫자를 할당하는 것이 의미적으로 옳음.
    
    - 문제 해결 능력을 통해 만든 해결 방안은 프로그래밍 언어의 문법을 사용해 표현 = 작성된 코드는 해결 방안의 구체적 구현물
    => 프로그래밍 언어 문법에 부합하는 것은 물론, 요구사항이 실현(문제가 해결)되어야 의미가 있음.

    ** 결국 프로그래밍이란?
    - 요구사항의 집합을 분석해서 적절한 자료구조와 함수의 집합으로 변환 후, 흐름을 제어하는 것
